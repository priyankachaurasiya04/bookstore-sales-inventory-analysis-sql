# 📚 Bookstore Sales and Inventory Analysis Using SQL

## 📌 Project Overview

This project focuses on analyzing bookstore operations using SQL to extract meaningful business insights from books, customers, and orders data. The analysis helps understand sales performance, inventory status, customer purchasing behavior, and revenue generation.

## 🎯 Objectives

* Analyze book inventory and stock availability.
* Identify customer purchasing patterns.
* Evaluate order trends and revenue performance.
* Generate business insights using SQL queries.
* Strengthen SQL skills through real-world business scenarios.

## 🛠️ Tools & Technologies

* PostgreSQL

## 🗂️ Database Tables

### Books

Contains information about books such as:

* Book_ID
* Title
* Author
* Genre
* Published_Year
* Price
* Stock

### Customers

Contains customer details:

* Customer_ID
* Name
* Country
* City

### Orders

Contains order transaction details:

* Order_ID
* Customer_ID
* Book_ID
* Order_Date
* Quantity
* Total_Amount

## 📊 Analysis Performed

### Basic SQL Queries

* Retrieve books by genre.
* Find books published after a specific year.
* Filter customers by country.
* Analyze orders within a date range.
* Calculate total stock available.
* Identify the most expensive book.
* Find orders with quantity greater than one.
* Retrieve high-value orders.
* List all available genres.
* Find books with the lowest stock.
* Calculate total revenue.

### Advanced SQL Queries

* Genre-wise books sold analysis.
* Average price analysis by genre.
* Identify repeat customers.
* Find the most frequently ordered book.
* Top 3 expensive fantasy books.
* Author-wise quantity sold.
* Cities with high-spending customers.
* Highest spending customer analysis.
* Remaining inventory calculation after fulfilling orders.

## 🔍 Key Insights

* Revenue can be tracked effectively using order data.
* Customer purchase behavior helps identify loyal customers.
* Inventory analysis highlights low-stock books requiring attention.
* Genre-wise sales provide insights into customer preferences.
* Remaining stock calculations support inventory planning.
